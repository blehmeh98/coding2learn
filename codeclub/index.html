Introduction

We’re going to make a game that tests your reflexes. The two sprites on the screen are going to change repeasedly and the player will have to stop the sprites when they match by using the spacebar.

STEP 1: Create a sprite that can change costumes.

Let’s make a sprite that has several costumes.

Activity Checklist

  1. Start a new Scratch project.
  2. Delete the Cat sprite and create a new one. You can select any costume you like. In this example we use things/a-block.
  3. Change the name of the sprite to 'LeftSprite' by clicking on the blue i symbol.
  4. Now you'll need to add some extra costumes. Click on the Costumes tab, and add in three other costumes. We've chosen the letter blocks from things, but you can choose whatever you like.

Step2: Switching costumes

Now let's make the costumes change automatically.

Activity Checklist.

  1. Click the Scripts tab
  2. Click Events and drag a when flag clicked into the scripts area. This will be triggered when we click the green flag.
  3. Create a new variable by clicking Data and Make a variable. Call it leftCostume, then uncheck the box next to it so it doesn’t display on the stage.
  4. Drag the Set leftCostume block into the scripts area
  5. Now we want to pick a random number from 1 to 4 and set the costume to that number. Click on the Operators tab and then drag the pick random 1 to 10 block into the scripts area. Change the 10 to a 4. Then drag the pick random 1 to 4 into the Set leftCostume to block

You should see the Sprite change costume.

We want the player to be able to stop the costume change by pressing the spacebar

Activity Checklist

  2. Now, use the gameOn variable we created earlier and set up the block so that it reads gameOn=False
  3. Now you can place this block into the repeat until loop you created earlier.
  4. Now the costume will keep changing as long as gameOn is True. All we need to do now is change gameOn to False when the spacebar is pressed. Click on Events and drag the when spece key pressed into the scripts area. Underneath it place a set gameOn to False.

STEP6:Setting up the Stage

Let's set up the stage to let the player know if they've won or lost.

Activity Checklist

  1. Click on the Stage object. We're going to want a few backdrops. Click on the Backdrops tab and choose a backdrop. We're using other/rays.
  2. Now rename the backdrop to 'Game'.
  3. Duplicate the backdrop and use the text tool to write the words 'You Lose' on the screen.
  4. Duplicate the second backdrop and edit the words to 'You Win'. Rename the backdrops so they are called 'Lose' and 'Win'.
  5. The first thing we want, is to always reset the backdrop to 'Game' when the player starts. Drag in a When greenflag clicked block into the Stage's script area. Underneath it add a switch backdrop to Game block.

Test Your Project

Click the green flag and make sure that the stage is set to Game.

Save your project

STEP 7: Deciding if you've won or lost.

Now we need to see if the player has won or lost.

Activity Checklist

  1. We'll need the game to be always listening out to see if the player has ended the game or not. For this we'll use a forever loop. Drag one in and place it below the switch Backdrop to Game block.
  2. Now we'll need to test to see if the game is over or not. We can use an if block for this. Find it in the Control section and drag it beneath your script.
  3. The if statement will test to see if gameOn = False. See if you can build that, and place it in the if block.
  4. Next you'll need an if...else... block. This is called a conditional. Find it in the Control palate and drag it into your if block.
  5. We know that the blocks will match if the variables leftCostume and rightCostume are the same. See if you can build a block that reads rightCostume = leftCostume, and place it into the if...else... block.
  6. Just to finish off we want to change the backdrop of the stage. If rightCostume = leftCostume then the stage should switch backdrop to Win. Otherwise it should switch to switch backdrop to Lose
